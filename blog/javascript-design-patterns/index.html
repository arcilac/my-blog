JavaScript Design Patterns

Highlights from Addy Osmani's Book

What is a Design Pattern?

A design pattern is a reusable solution to a common problem in software development. In his book Learning JavaScript Design Patterns, Addy Osmani introduces some of the most useful and relevant patterns for JavaScript development.

This summary focuses on the most interesting and practical patterns you can start using today.

1. Module Pattern

This pattern helps encapsulate code and keep variables private using Immediately Invoked Function Expressions (IIFEs).

Example:

const counter = (function() {
  let count = 0;

  return {
    increment() { count++; },
    getCount() { return count; }
  };
})();

Benefits:

  • Encapsulates state
  • Avoids polluting the global scope
  • Enables creation of public and private APIs

2. Revealing Module Pattern

A variant of the module pattern where all functions and variables are defined in the private scope and the return statement explicitly reveals the ones to expose.

Example:

const calculator = (function() {
  function add(a, b) { return a + b; }
  function subtract(a, b) { return a - b; }

  return {
    add,
    subtract
  };
})();

Benefits:

  • Improves code readability
  • Clearly separates public and private members

3. Singleton Pattern

Ensures that a class has only one instance and provides a global point of access to it.

Example:

const Singleton = (function () {
  let instance;

  function createInstance() {
    return { time: new Date() };
  }

  return {
    getInstance() {
      if (!instance) {
        instance = createInstance();
      }
      return instance;
    }
  };
})();

When to Use:

  • Managing global configurations
  • When only one shared resource should exist (e.g., database connection)

4. Observer Pattern

Allows multiple objects to listen to and react to events emitted by another object.

Simple Example:

class EventEmitter {
  constructor() {
    this.events = {};
  }

  on(event, listener) {
    if (!this.events[event]) this.events[event] = [];
    this.events[event].push(listener);
  }

  emit(event, data) {
    if (this.events[event]) {
      this.events[event].forEach(listener => listener(data));
    }
  }
}

Use Cases:

  • Reactive interfaces
  • Custom event systems

5. Factory Pattern

Creates objects without exposing the creation logic to the client. Useful when the type of object to be created can vary dynamically.

Example:

function Dog(name) {
  this.name = name;
}

function Cat(name) {
  this.name = name;
}

function AnimalFactory() {}

AnimalFactory.prototype.createAnimal = function(type, name) {
  switch(type) {
    case 'dog': return new Dog(name);
    case 'cat': return new Cat(name);
  }
};

const factory = new AnimalFactory();
const fido = factory.createAnimal('dog', 'Fido');

Quick Pattern Comparison

PatternPurposeWhen to Use
ModuleEncapsulate and organize codeWhenever private variables are needed
Revealing ModuleClearly expose only what’s neededFor clearer public APIs
SingletonSingle shared instanceGlobal config or unique resources
ObserverEvent-based communicationInterfaces or pub/sub systems
FactoryDynamically create objectsWhen multiple object types are needed

Conclusion

Design patterns are not strict rules but powerful tools. Understanding how and when to use them, as shown in Addy Osmani's book, can make your code cleaner, more scalable, and easier to maintain.
