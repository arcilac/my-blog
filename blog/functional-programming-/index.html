Functional programming patterns

Pure functions, immutability, and composition

What is functional programming?

Functional Programming (FP) is a programming paradigm centered around the use of pure functions, immutability, and function composition. It promotes writing predictable, reusable, and testable code by avoiding shared state and side effects.

FP is especially popular in JavaScript, where functions are first-class citizens and can be passed, returned, or composed freely.

Pure functions

A pure function is one that:

  • Always returns the same output given the same input
  • Has no side effects (does not modify external variables, DOM, or global state)

Example:

function add(a, b) {
  return a + b;
}

This function is pure because it doesn't rely on or modify anything outside itself.

Benefits:

  • Easier to test
  • Predictable behavior
  • Reusable in different contexts

Immutability

Immutability means that once a value is created, it cannot be changed. Instead of modifying existing data, we create new copies with the desired changes.

Example:

const numbers = [1, 2, 3];
const newNumbers = [...numbers, 4];

We use the spread operator to create a new array without mutating the original.

Why it matters:

  • Prevents unexpected side effects
  • Makes state management easier (e.g., in React)

Function composition

Composition is combining smaller functions to build more complex ones. It's about creating pipelines where the output of one function is the input of another.

Example:

const double = x => x * 2;
const square = x => x * x;

const doubleThenSquare = x => square(double(x));

You can also use libraries like lodash/fp or write your own compose or pipe functions to enhance this pattern.

Key concepts comparison

ConceptDescriptionBenefit
Pure FunctionsFunctions without side effectsTestable, predictable
ImmutabilityData doesn't change; create copies insteadAvoid bugs from shared state
CompositionCombine simple functions into more complex behaviorReusability, cleaner abstractions

Best practices

  • Avoid mutating parameters or shared variables
  • Favor small, single-purpose functions
  • Use utility libraries like Ramda or Lodash/fp for helpers
  • Prefer map, filter, reduce over for loops for transformations

Conclusion

Functional programming patterns like pure functions, immutability, and composition help you write more reliable, maintainable, and scalable code. These principles are especially useful in large codebases and frameworks like React where state management and predictable behavior are crucial.
