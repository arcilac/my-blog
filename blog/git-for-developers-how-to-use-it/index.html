Git for developers: how to use it with real examples

When you're working on any software project—whether it's a solo portfolio site or a team-based enterprise app—Git is an essential tool. It allows you to track changes, collaborate with others, and prevent disaster when things go wrong.

In this article, we’ll go beyond theory and show you how to actually use Git in your development workflow, with real commands and use cases.

🧩 What is git?

Git is a distributed version control system. It tracks changes in your code over time, lets you experiment safely, and enables collaboration across teams.

In simpler terms:

  • You can save versions of your project (commits)
  • You can go back in time if something breaks
  • You can collaborate without overwriting others’ work

All of this from the command line or tools like VS Code, GitHub Desktop, or GitKraken.

🚀 Basic git workflow

1. Initialize a git repo

git init

Creates a .git folder to start tracking your code.

2. Stage and commit changes

git add .
git commit -m "Initial commit"

This saves a snapshot of your code.

3. Link to a remote (e.g., github)

git remote add origin https://github.com/user/repo.git
git push -u origin main

✨ Useful git commands (with context)

CommandWhat it does
git statusShows what's changed, staged, or untracked
git log --onelineView commit history
git diffSee what’s changed but not committed
git branchShow or create branches
git checkout -b feature-xCreate and switch to a new branch
git merge feature-xMerge a branch into your current one
git pull origin mainPull latest changes from remote
git push origin mainPush your changes to GitHub

👥 Collaborating with others

When working with a team:

  1. Clone the repo
git clone https://github.com/team/project.git
  1. Create a new branch for your feature
git checkout -b fix-login-bug
  1. Make changes and commit them
git add .
git commit -m "Fix login validation bug"
  1. Push your branch and open a pull request
git push origin fix-login-bug
  1. Review, merge, and delete the branch when done

🧪 Fixing mistakes

Made a typo in your commit message?

git commit --amend

Need to undo your last commit?

git reset --soft HEAD~1

Accidentally deleted something and want it back?

git checkout -- path/to/file

Want to roll back to a stable commit?

git reset --hard <commit-hash>

🔄 Branching and merging strategy

Use branches to separate work:

  • main: production-ready code
  • dev: staging or testing
  • feature/*: new features
  • bugfix/*: quick fixes

Use pull requests (PRs) to:

  • Trigger code reviews
  • Run automated tests
  • Keep the main branch clean

🗃️ .Gitignore and keeping your repo clean

Create a .gitignore file to avoid tracking unnecessary files:

node_modules/
.env
.DS_Store
dist/

Use gitignore.io to generate it based on your stack.

🔐 Keeping secrets out of git

Never commit .env or secret API keys. Use .gitignore, and for shared secrets, consider tools like:

  • GitHub Actions secrets
  • Doppler, 1Password, or Vault
  • Environment variable management

🧠 Summary checklist

✅ Use git init, add, and commit for local tracking

✅ Create branches for new features or fixes

✅ Use pull and push to sync with remote repos

✅ Don’t commit secret files—use .gitignore

✅ Use PRs to merge changes collaboratively

✅ Use git log, diff, and status to debug your history

🧠 Conclusion

Git isn’t just for teams or large projects—it’s a personal safety net for your code. Whether you’re experimenting, collaborating, or deploying, Git helps you stay organized, safe, and efficient.

If you're not using Git yet, now is the time. And if you already are—learn one new command today and improve your workflow even more.
