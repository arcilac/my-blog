Websockets

Real-time communication in the browser

What are websockets?

WebSockets are a communication protocol that allows for full-duplex, persistent connections between a client (like a browser) and a server. Unlike HTTP, which is request-response based, WebSockets allow both the client and server to send messages at any time without needing to reopen a connection.

Why use websockets?

WebSockets shine in applications that require real-time updates. Common use cases include:

  • Chat applications
  • Live sports scores
  • Online gaming
  • Collaborative editing tools
  • Financial dashboards (e.g., stock tickers)

How do they work?

When a WebSocket connection is initiated:

  1. The client sends an HTTP request to upgrade the connection.
  2. The server accepts and switches the protocol to WebSocket.
  3. A persistent, bidirectional communication channel is established.

Example (client side):

const socket = new WebSocket('wss://example.com/socket');

socket.onopen = () => {
  console.log('Connected to server');
  socket.send('Hello from client!');
};

socket.onmessage = (event) => {
  console.log('Message from server:', event.data);
};

socket.onclose = () => {
  console.log('Disconnected');
};

Benefits of websockets

FeatureBenefit
Persistent ConnectionReduces overhead of repeated handshakes
Full-DuplexServer and client can push messages independently
Low LatencyIdeal for real-time apps
Event-DrivenNaturally integrates with reactive programming

When not to use websockets

While powerful, WebSockets may not be ideal for every scenario:

  • If your app is mostly read-only and doesn’t require real-time updates.
  • If your infrastructure or security policies restrict persistent connections.
  • If HTTP/2 with Server-Sent Events (SSE) is sufficient for your needs.

Alternatives

TechnologyDescriptionBest Use
HTTP PollingFrequent HTTP requests for updatesSimple, less efficient
Server-Sent Events (SSE)One-way streaming from server to clientLightweight push
WebRTCPeer-to-peer data/audio/videoVideo calls, direct file sharing

Conclusion

WebSockets empower developers to create interactive, real-time web applications. While they may not be necessary for every project, they are essential when your app demands low-latency communication between the client and the server.
