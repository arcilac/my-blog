Zustand and CDNs

Efficient and fast global state in react applications

What is zustand?

Zustand is a minimalist global state management library for React, created by the authors of Jotai and React Three Fiber.

It is simple, scalable, and doesn’t require boilerplate or the Context API.

Why use zustand?

BenefitExplanation
Minimalist APIUses simple hooks (useStore) without providers or context
Fast and reactiveNo unnecessary re-renders, only updates what changed
Easy to scaleIdeal for both small and large apps with modular structures
SSR (Server-side Rendering)Can be used in server-side rendered apps
Useful middlewareSupports persistence, subscriptions, and more

Basic example

import create from 'zustand'

const useStore = create((set) => ({
  count: 0,
  increase: () => set((state) => ({ count: state.count + 1 })),
}))

export default function Counter() {
  const { count, increase } = useStore()
  return (
    <div>
      <h1>{count}</h1>
      <button onClick={increase}>Increment</button>
    </div>
  )
}

What is a CDN?

A Content Delivery Network (CDN) is a distributed network of servers that deliver web resources like JavaScript, CSS, images, or even entire libraries.

It ensures that files are served from geographically closer locations to the user.

Why use a CDN with zustand (and other libraries)?

  • ✅ Faster loading: Reduced latency for the end user.
  • ✅ Better caching: Popular static files are stored in the browser.
  • ✅ Lower server load: The client fetches resources directly from the CDN network.
  • ✅ Use in environments without installation: You can use Zustand in demos or sandboxes without npm install.

Example of using zustand from a CDN (with a bundler)

<script src="https://unpkg.com/zustand/umd/zustand.min.js"></script>
<script>
  const { create } = window.zustand
  const useStore = create((set) => ({
    count: 0,
    increase: () => set((state) => ({ count: state.count + 1 })),
  }))
</script>
> You can also load it from jsDelivr or other providers.

Best practices

  • Use Zustand when you need a lightweight global state solution without the complexities of Redux.
  • Use a CDN in production to serve libraries if you’re in static environments or without modern bundlers.
  • For modern React projects, Zustand is best installed via npm or pnpm.

Conclusion

Zustand is an elegant and powerful alternative for state management in React applications.

By combining it with a CDN, you can achieve ultra-fast loading and a lightweight architecture, perfect for prototypes, testing, or production applications with distributed resources.
