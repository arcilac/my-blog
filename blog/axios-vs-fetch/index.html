Axios vs fetch

If you've built JavaScript apps, you've probably made HTTP requests. While the browser's native fetch() works, most developers reach for Axios instead. But why? What makes Axios so popular?

üåê What is axios?

Axios is a promise-based HTTP client for JavaScript that works in both browsers and Node.js. It's essentially a wrapper around HTTP requests that makes them simpler, more reliable, and feature-rich.

Think of it as fetch() with superpowers.

// Basic Axios request
import axios from 'axios';

const response = await axios.get('https://api.example.com/users');
console.log(response.data);

That's it. Clean, simple, and it just works.

üìå Why choose axios over fetch?

While fetch() is native and doesn't require installation, Axios provides several advantages:

‚úÖ Automatic JSON parsing

// Fetch (more verbose)
const response = await fetch('/api/users');
const data = await response.json();

// Axios (cleaner)
const { data } = await axios.get('/api/users');

‚úÖ Better error handling

// Fetch doesn't reject on 4xx/5xx
const response = await fetch('/api/users');
if (!response.ok) {
  throw new Error('Request failed');
}

// Axios automatically rejects on error status
try {
  const { data } = await axios.get('/api/users');
} catch (error) {
  console.log('Request failed:', error.message);
}

‚úÖ Request/response interceptors

// Add auth token to all requests
axios.interceptors.request.use(config => {
  config.headers.Authorization = `Bearer ${getToken()}`;
  return config;
});

üöÄ Common axios patterns

GET Requests

// Simple GET
const users = await axios.get('/api/users');

// GET with query parameters
const users = await axios.get('/api/users', {
  params: { page: 1, limit: 10 }
});

POST Requests

// Create new user
const newUser = await axios.post('/api/users', {
  name: 'Marta Perez',
  email: 'Marta@example.com'
});

// POST with custom headers
const response = await axios.post('/api/users', userData, {
  headers: { 'Content-Type': 'application/json' }
});

PUT/PATCH/DELETE

// Update user
await axios.put(`/api/users/${userId}`, updatedData);

// Partial update
await axios.patch(`/api/users/${userId}`, { name: 'New Name' });

// Delete user
await axios.delete(`/api/users/${userId}`);

‚öôÔ∏è Advanced axios features

Creating instances

const api = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 5000,
  headers: {
    'Content-Type': 'application/json'
  }
});

// Use the instance
const users = await api.get('/users');

Request/response interceptors

// Request interceptor (add auth)
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Response interceptor (handle errors)
api.interceptors.response.use(
  response => response,
  error => {
    if (error.response.status === 401) {
      // Redirect to login
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

Concurrent requests

const [users, posts, comments] = await Promise.all([
  axios.get('/api/users'),
  axios.get('/api/posts'),
  axios.get('/api/comments')
]);

üí° Best practices

‚úÖ Use instances for different APIs

const authAPI = axios.create({ baseURL: '/auth' });
const dataAPI = axios.create({ baseURL: '/api/v1' });

‚úÖ Handle loading states

const [loading, setLoading] = useState(false);

const fetchData = async () => {
  setLoading(true);
  try {
    const { data } = await axios.get('/api/data');
    setData(data);
  } catch (error) {
    setError(error.message);
  } finally {
    setLoading(false);
  }
};

‚úÖ Use abortcontroller for cancellation

useEffect(() => {
  const controller = new AbortController();

  const fetchData = async () => {
    try {
      const { data } = await axios.get('/api/data', {
        signal: controller.signal
      });
      setData(data);
    } catch (error) {
      if (!axios.isCancel(error)) {
        setError(error.message);
      }
    }
  };

  fetchData();

  return () => controller.abort();
}, []);

‚úÖ Summary checklist

  • ‚úÖ Axios simplifies HTTP requests with automatic JSON parsing
  • ‚úÖ Better error handling than native fetch()
  • ‚úÖ Interceptors let you modify requests/responses globally
  • ‚úÖ Instances help organize different API endpoints
  • ‚úÖ Works in both browser and Node.js environments
  • ‚úÖ Great for building reusable API clients

üß† Conclusion

Axios has become the go-to HTTP client for JavaScript developers because it just works. While fetch() is getting better, Axios still provides a superior developer experience with its rich feature set and intuitive API.

Whether you're building a React app, Node.js backend, or any JavaScript project that needs HTTP requests, Axios will make your life easier. The automatic JSON parsing, error handling, and interceptors alone are worth the small bundle size.

Next time you need to make HTTP requests, give Axios a try. Your future self will thank you.
