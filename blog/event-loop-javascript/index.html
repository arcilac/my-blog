The event loop in javascript

How it works

Introduction to the event loop

JavaScript is a single-threaded programming language, meaning it can only execute one task at a time. However, thanks to the Event Loop, it can handle asynchronous operations such as file reading, HTTP requests, and timers without blocking code execution.

The Event Loop is the mechanism that allows JavaScript to execute tasks efficiently without stopping the execution of other functions.

How does the event loop work?

The Event Loop follows an execution cycle consisting of the following elements:

  1. Call Stack: A LIFO (Last In, First Out) structure where functions currently being executed are stored.
  2. Web APIs: Functions provided by the browser (such as setTimeout, fetch, DOM events).
  3. Task Queue: Stores functions that should be executed once the call stack is empty.
  4. Microtask Queue: Contains resolved promises and process.nextTick in Node.js, and has priority over the task queue.

Event loop cycle:

  1. Executes functions in the Call Stack.
  2. If an asynchronous function (such as setTimeout) is called, it is delegated to Web APIs.
  3. Once the asynchronous function completes, its callback is placed in the Task Queue or Microtask Queue.
  4. If the Call Stack is empty, the Event Loop takes tasks from the Microtask Queue (with priority) and then from the Task Queue.

Practical example of the event loop

console.log('Start');

setTimeout(() => {
  console.log('Timeout executed');
}, 0);

Promise.resolve().then(() => {
  console.log('Promise resolved');
});

console.log('End');

Expected output:

Start
End
Promise resolved
Timeout executed

Explanation:

  1. console.log('Start') executes first.
  2. setTimeout is delegated to Web APIs, and its callback is sent to the Task Queue.
  3. Promise.resolve().then(...) is added to the Microtask Queue.
  4. console.log('End') executes.
  5. Tasks in the Microtask Queue (promises first) execute.
  6. Tasks in the Task Queue (setTimeout) execute.

Conclusion

The Event Loop is fundamental to understanding how JavaScript handles asynchrony. Knowing how it works will help you optimize your applications' performance and avoid unnecessary code blocking.
