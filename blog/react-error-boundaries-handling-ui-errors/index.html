React Error Boundaries

Catching and Handling UI Errors Gracefully

What Are Error Boundaries?

Error boundaries are a React-specific mechanism for handling errors that occur during rendering, in lifecycle methods, or in constructors of the component tree. They help prevent the entire app from crashing by allowing you to catch errors and display a fallback UI.

How Do Error Boundaries Work?

An error boundary is a React component that implements either or both of these lifecycle methods:

  • static getDerivedStateFromError(error)
  • componentDidCatch(error, info)

These methods let the component detect and respond to rendering errors in its child component tree.

Basic Example

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    console.error("Error caught by boundary:", error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return <h2>Something went wrong.</h2>;
    }

    return this.props.children;
  }
}

// Usage
<ErrorBoundary>
  <MyComponent />
</ErrorBoundary>

Why Use Error Boundaries?

BenefitExplanation
Prevent UI crashesCatches errors in part of the UI without affecting the rest
Improve user experienceDisplay fallback instead of a broken app
Debugging supportLog or report errors for easier debugging

Important Notes

  • Error boundaries only catch errors in child components during rendering, lifecycle methods, and constructors.
  • They do not catch:
  • Event handlers (you need try/catch for those)
  • Asynchronous code like setTimeout or fetch
  • Server-side rendering errors
  • Errors thrown in the error boundary itself

Best Practices

  • Use error boundaries around critical parts of your UI (like routes, widgets, or dashboards).
  • Create multiple error boundaries if needed, so one component failing doesn’t bring down the whole app.
  • Log errors to monitoring services (e.g., Sentry, LogRocket).

Conclusion

React Error Boundaries provide a powerful way to catch rendering errors and keep your UI resilient. By isolating error-prone components and providing fallback UIs, you improve stability, user experience, and debugging processes in your React applications.
