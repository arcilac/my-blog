Currying in JavaScript

What It Is and How It Works

Introduction to Currying

Currying is a functional programming technique where a function is transformed into a sequence of functions, each taking a single argument. Instead of calling a function with multiple parameters, currying allows calling a series of nested functions, each taking one argument at a time.

This technique improves code reusability and helps create more flexible and readable functions.

How Does Currying Work?

In JavaScript, currying can be implemented by returning a function for each argument until all required arguments are received. Once all arguments are provided, the final function executes the logic.

Example Without Currying:

function multiply(a, b) {
  return a * b;
}

console.log(multiply(2, 3)); // Output: 6

Example With Currying:

function curriedMultiply(a) {
  return function (b) {
    return a * b;
  };
}

const double = curriedMultiply(2);
console.log(double(3)); // Output: 6
console.log(double(5)); // Output: 10

Here, curriedMultiply(2) returns a new function that multiplies any number by 2.

Advantages of Currying

  1. Code Reusability: Allows creating specialized functions from a general function.
  2. Readability and Maintainability: Makes the code easier to understand and modify.
  3. Functional Programming Paradigm: Encourages the use of pure functions and avoids side effects.
  4. Partial Application: Functions can be preconfigured with certain arguments and reused efficiently.

Implementing a Generic Currying Function

We can create a reusable currying function that works with any function:

function curry(func) {
  return function curried(...args) {
    if (args.length >= func.length) {
      return func.apply(this, args);
    }
    return (...nextArgs) => curried(...args, ...nextArgs);
  };
}

function add(a, b, c) {
  return a + b + c;
}

const curriedAdd = curry(add);
console.log(curriedAdd(1)(2)(3)); // Output: 6

Conclusion

Currying is a powerful technique in JavaScript that improves function flexibility and promotes cleaner code. Understanding and using currying effectively can help you write more modular and reusable code in functional programming.
