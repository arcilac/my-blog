Geolocation in Web Apps: How to Use It (and When You Should)

Have you ever used an app that shows your location, suggests nearby stores, or personalizes content based on where you are?

That’s geolocation in action—and it’s easier to implement than you might think.

🌐 What Is Geolocation?

Geolocation is the ability of a web app to detect a user’s location using the browser. The most common API for this is the Geolocation API, built into most modern browsers.

> It can use GPS, Wi-Fi, IP address, or cell towers to approximate location.

📦 Common Use Cases

  • 🗺️ Show user location on a map
  • 🛒 Suggest nearby stores or services
  • 🍔 Auto-fill delivery address
  • 🎯 Target content by region (e.g. language or promotions)
  • 🚀 Trigger geo-based automations (like alerts or analytics)

📍 Getting Location in JavaScript

Here’s how to get a user’s coordinates with the Geolocation API:

navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log(position.coords.latitude, position.coords.longitude);
  },
  (error) => {
    console.error('Error getting location:', error);
  }
);

📌 Output:

latitude: 40.7128
longitude: -74.0060
> ⚠️ The browser will ask for permission before giving you the location.

⚛️ Using Geolocation in React

Here’s a simple React hook for geolocation:

import { useState, useEffect } from 'react';

export const useGeolocation = () => {
  const [coords, setCoords] = useState(null);
  const [error, setError] = useState(null);

  useEffect(() => {
    navigator.geolocation.getCurrentPosition(
      (pos) => setCoords(pos.coords),
      (err) => setError(err)
    );
  }, []);

  return { coords, error };
};

Usage:

const { coords, error } = useGeolocation();

if (coords) {
  console.log('Latitude:', coords.latitude);
}

🛰️ Real-Time Location with watchPosition

If you want to track movement (e.g., delivery driver or user walking), use watchPosition:

const watchId = navigator.geolocation.watchPosition((pos) => {
  console.log('Updated coords:', pos.coords);
});

To stop watching:

navigator.geolocation.clearWatch(watchId);

🔒 Privacy & Permission

  • Users must explicitly allow location access.
  • Browsers may block access on non-HTTPS pages.
  • Always provide fallbacks and transparent UX.

❗ Always ask:

> “Does this feature really need their location?”

🗺️ Showing Location on a Map

You can combine geolocation with map libraries like:

  • 🧭 Leaflet + React Leaflet
  • 🗺️ Google Maps API
  • 🌐 Mapbox

Example with React Leaflet:

<Map center={[coords.latitude, coords.longitude]} zoom={13}>
  <Marker position={[coords.latitude, coords.longitude]} />
</Map>

🧪 Common Errors to Handle

{
  code: 1, // PERMISSION_DENIED
  code: 2, // POSITION_UNAVAILABLE
  code: 3  // TIMEOUT
}

Handle errors gracefully with a message or fallback UI.

✅ Checklist

  • ✅ Use getCurrentPosition() for one-time location
  • ✅ Use watchPosition() for real-time tracking
  • ✅ Always handle permission errors
  • ✅ Require HTTPS for production use
  • ✅ Pair with maps for a great user experience

🧠 Conclusion

Geolocation adds context to your web apps and unlocks powerful features—from personalized content to real-time maps.

But with great power comes great responsibility: always use location data ethically, securely, and with user consent.
