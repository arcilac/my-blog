JWT

Secure and Scalable Authentication for Web Applications

What is JWT (JSON Web Token)?

JWT is an open standard (RFC 7519) that defines a compact and self-contained way to securely transmit information between parties as a JSON object. It is widely used for authentication and authorization in modern applications.

How Does JWT Work?

A JWT consists of three parts, separated by dots:

  1. Header: specifies the signing algorithm and token type.
  2. Payload: contains the data (claims), such as userId, role, or email.
  3. Signature: created by signing the header and payload with a secret key.
HEADER.PAYLOAD.SIGNATURE

The token is generated on the backend and sent to the client (usually during login). Then, the client includes it in every subsequent request (typically in the Authorization header), allowing the backend to verify the user without storing session data.

Advantages of Using JWT

FeatureBenefit
StatelessNo need to store sessions on the server
ScalableIdeal for distributed systems and microservices
CompactEasy to transmit, even in HTTP headers
Self-containedAll necessary information is inside the token

Basic Example in Node.js with Express

const jwt = require('jsonwebtoken');

// Create a token
const token = jwt.sign({ userId: 123 }, 'my_secret_key', { expiresIn: '1h' });

// Verify the token
jwt.verify(token, 'my_secret_key', (err, decoded) => {
  if (err) return res.sendStatus(403);
  console.log(decoded.userId);
});

Best Practices

  • Always use HTTPS to prevent Man-In-The-Middle attacks.
  • Set a short expiration (expiresIn) to limit risk.
  • Store the token safely: ideally in memory, not localStorage if avoidable.
  • Rotate and revoke tokens if misuse is suspected.

When _Not_ to Use JWT?

JWTs are powerful, but not ideal in every scenario:

  • If you need to frequently invalidate sessions (JWT makes this hard without a blacklist).
  • If your app is simple and could benefit from traditional server-side sessions.

Conclusion

JWT has transformed the way we handle web authentication. It provides a secure, scalable, and stateless mechanism to authenticate users and protect endpoints. Like any tool, understanding its use cases and limitations ensures you implement it effectively.
