How to use environment variables correctly in frontend and backend

Environment variables are essential for modern web development. They allow you to configure your app for different environments, development, staging, production, without changing the source code.

But while they're easy to use in backend code, they can be misused or misunderstood in frontend applications.

Frontend code runs in the browser, so not all env vars are safe to expose.

If you use tools like Vite, Create React App, or Next.js, only environment variables prefixed with a special string are included in the client-side bundle.

Any env var used in the frontend is visible to anyone who opens DevTools.

‚úÖ OK:

  • VITE_API_URL
  • REACT_APP_ANALYTICS_ID

‚ùå NOT OK:

  • VITE_SECRET_KEY
  • REACT_APP_DB_PASSWORD

üõ°Ô∏è Protecting secrets

‚úÖ Keep secrets server-side only

For example, in a full-stack app:

  • Use VITE_API_URL in the frontend to talk to your own backend
  • Keep the SECRET_KEY, DB credentials, and third-party tokens in the backend .env

If your frontend needs to interact with a third-party API, proxy it through your backend if authentication is required.

üì¶ Managing env files

Recommended file structure:

.env.development # Local dev config
.env.production  # Production overrides
.env.local       # Machine-specific config (not in Git)

üßº Git ignore

Always add .env.local and sensitive .env.* files to your .gitignore:

# .gitignore
.env.local
.env.*.local

‚úÖ Summary

  • Use environment variables to keep config out of your code
  • Backend: use dotenv, validate vars, never hardcode
  • Frontend: only expose non-sensitive values with the correct prefix
  • Never expose secrets in the browser
  • Use .env, .env.local, and .gitignore correctly

üîö Final thoughts

Environment variables are a simple but powerful tool. When used correctly, they keep your apps secure, portable, and configurable, whether you're deploying to Vercel, Docker, or bare metal.

Just remember:

If it's used in the frontend, it's public.

If it's sensitive, keep it on the server.
