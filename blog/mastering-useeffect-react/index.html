A side effect is any operation that interacts with the outside world or causes a change outside the current function's scope.

Examples include:

  • Fetching data from an API

  • Writing to local storage
  • Setting timers

In React, side effects should not run during rendering‚Äîthat‚Äôs where useEffect comes in.

‚úÖ The empty array means the effect runs once after the initial render‚Äîlike componentDidMount.

üåê Fetching data with useeffect

function UsersList() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    async function fetchUsers() {
      const res = await fetch('/api/users');
      const data = await res.json();
      setUsers(data);
    }

    fetchUsers();
  }, []);

  return (
    <ul>
      {users.map(u => (
        <li key={u.id}>{u.name}</li>
      ))}
    </ul>
  );
}

üìù Tips:

  • Use async functions inside the effect.
  • Always provide a dependency array to avoid unnecessary requests.

üîÅ Useeffect with dependencies

This effect runs every time value changes.

You can include multiple dependencies:

  console.log('User or token changed');
}, [user, token]);
